# limonesska.ru

Личный блог-дневник «Моя броня» — **limonesska.ru**. Astro 6, деплой на Vercel, контент на русском. Растёт по чешуйке: статьи, заметки, дневник, библиотека, лунный календарь.

## 🧞 Команды

Все команды запускаются из корня проекта:

| Команда | Действие |
| :--- | :--- |
| `npm install` | Установить зависимости |
| `npm run dev` | Локальный dev-сервер на `localhost:4321` |
| `npm run build` | Прод-сборка в `./dist/` |
| `npm run preview` | Предпросмотр прод-сборки локально |
| `npm run upload-photos` | Залить фото дневника из `photos-upload/` в Cloudinary и получить URL |

Тестов и линтера нет.

> ⚠️ После изменения `content.config.ts` (коллекции) перезапусти dev-сервер — новые коллекции на лету не подхватываются.

## 🗂 Структура

```text
src/
├── layouts/
│   ├── BaseLayout.astro    # <head>, дизайн-токены :root (+ тёмная тема), Header + main + Footer
│   └── PostLayout.astro    # шаблон поста/заметки (.prose, TOC, сноски, ссылки, бэклинки)
├── components/
│   ├── Header.astro        # шапка, меню-дропдаун, переключатель темы
│   ├── Footer.astro        # подвал
│   ├── Icon.astro          # инлайн SVG-иконок (currentColor)
│   ├── PostCard.astro      # карточка статьи (главная, /blog)
│   ├── EntryList.astro     # общий список для страниц-фильтров
│   └── Photo.astro         # инлайн-фото в теле записей (size/align/caption)
├── content/
│   ├── blog/               # статьи (.mdx)
│   ├── notes/              # заметки (.mdx)
│   └── diary/              # дневник — личные записи по дням (.mdx)
├── content.config.ts       # коллекции blog/notes/diary, общая схема
└── pages/
    ├── index, blog, notes, diary (календарь), now, category, subcategory, tags, search, 404
    ├── about, colophon, contacts, projects, tools, glossary   # о себе / сайте
    ├── library (Библиотека|Антибиблиотека), garden («Моя броня» — лента всех постов+заметок)
    ├── moon (лунный календарь — фазы/затмения/ретрограды/VoC на astronomy-engine)
    ├── og/[...slug].png.ts  # OG-превью для соцсетей (satori → PNG, на сборке)
    └── api/*                # serverless (лайки через Supabase), privacy, cookie
```

Сборочные скрипты — в `scripts/` (напр. `upload-photos.mjs` — заливка фото в Cloudinary).

Все страницы оборачиваются в `BaseLayout`. Шапку/подвал в страницах не дублировать.

## 📝 Контент

Посты — MDX в `src/content/blog/`, заметки — в `src/content/notes/`. Имя файла — kebab-case slug.

Frontmatter (обязательные `title`, `pubDate`, `tags`; остальное опционально):

```mdx
---
title: "Заголовок"
pubDate: 2026-05-28
tags: ["тег1", "тег2"]
description: "Короткое описание для списка"
category: "Рубрика"
subcategory: "Подрубрика"
stage: "base" | "advanced" | "max"   # иконка звезды-стадии
audience: "Для кого этот текст"        # блок «Предполагаемая аудитория»
cover: "https://..."                   # картинка для карточки
updatedDate: 2026-06-01                # дата обновления (опц.)
post: "<slug-поста>"                   # ТОЛЬКО для заметок: привязка к посту → блок «Заметки к этому посту»
verified: true | false                 # метка «проверено / не проверено лично» (нет поля = нет метки)
coming: true                           # анонс: в списках бейдж «Скоро», страница = заглушка «выйдет такого-то» (≠ draft)
draft: true                            # черновик: виден в dev, скрыт в прод-сборке
---
```

Оба раздела рендерятся через `PostLayout`. Спецэлементы (ссылки разных типов, сноски, TOC, буквица) пишутся прямо в теле как HTML — навешиваются автоматически. Демо со всеми элементами: `src/content/blog/ritualy-cifrovogo-sada.mdx`.

Страницы-фильтры агрегируют посты + заметки + дневник: `/tags/<тег>`, `/category/<рубрика>`, `/subcategory/<подрубрика>`.

**Дневник** (`src/content/diary/`) — отдельная коллекция личных записей по дням («memory»), тот же шаблон `PostLayout`. Страница `/diary` — месячный календарь (1 день = фото + запись): листание месяцев/годов, по клику выезжают запись (справа) и фото-галерея (слева), клик по фото → полноэкранный лайтбокс. Фото — поле `photos: [url, …]` (первое = главное в ячейке календаря); `title` у записи необязателен (бывают «фото-только» дни). Фото хранятся во внешнем хостинге (Cloudinary), в `photos:` — чистый URL, оптимизацию (формат/качество/размер) навешивает код. **В теле записи** фото вставляются инлайн компонентом `<Photo src={frontmatter.photos[N]} size="s|m|l" align="center|left|right|wide" caption="…" />` — отдельного фото-стрипа сверху нет.

## ✨ Доп. фичи

- **Лунный календарь** (`/moon`) — фазы Луны по дням (рисованные), «луна без курса», затмения и ретрограды планет; всё считается `astronomy-engine` (без внешних API), время по Москве. Есть печать года (месяц на страницу, A4).
- **Библиотека** (`/library`) — Библиотека\|Антибиблиотека с фильтрами по подрубрикам; **«Моя броня»** (`/garden`) — единая лента всех статей+заметок карточками.
- **OG-превью** — `src/pages/og/[...slug].png.ts` генерит картинки для соцсетей на сборке (satori → PNG, «ночное небо» с логотипом, заголовком, рубрикой/стадией). Мета — в `BaseLayout`/`PostLayout`.
- **SEO** — `@astrojs/sitemap` (`sitemap-index.xml`) + `public/robots.txt`.
- **Лайки** — глобальные, через Supabase (`src/pages/api/*`, serverless на Vercel).
- **«Популярное»** — центральная карточка на главной выбирается по лайкам: SSR показывает самый свежий пост, на клиенте переранжируется по `/api/likes` (у кого лайков больше — тот и популярный).

## 🎨 Стили и темы

- Весь CSS — инлайново в `.astro` через `<style is:global>`. Без Tailwind, CSS-модулей и отдельных файлов.
- Дизайн-токены (цвета, шрифты, размеры) — в `:root` в `BaseLayout.astro`. Всегда использовать токены, не сырые hex/px.
- **Светлая/тёмная тема:** 8 канонических цветовых токенов (`--bg`, `--text`, `--text-muted`, `--accent`, `--accent-2`, `--line`, `--card-bg`, `--shadow`); старые `--color-*` — алиасы на них. Тёмная — `html[data-theme="dark"]`. Переключатель (солнце/луна) в шапке, выбор в `localStorage`, применяется до отрисовки (без вспышки). Светлая по умолчанию.
- **Шрифты (3):** Playfair Display (`--font-display`), Vollkorn (`--font-vollkorn`), Manrope (`--font-ui`).
- **Код-блоки** — подсветка Shiki в две темы (`github-light`/`github-dark`, настройка в `astro.config.mjs`): светлая по умолчанию, тёмная через CSS-переменные по `data-theme`. Плашка по ширине содержимого (`fit-content`).

## 🤖 Разработка с ИИ

Сайт делается в духе живого цифрового сада — часть тех-экспериментов блога (вайб-кодинг, автоматизация). В разработке участвует **[Claude Code](https://claude.com/claude-code)**:

- **`CLAUDE.md`** — контекст проекта для агента: архитектура страниц, формат постов, дизайн-токены, правила стилей. Держит ответы агента в рамках принятого визуального языка.
- **Бортовой журнал** (`CHANGELOG.md`, локальный, в `.gitignore`) — лог работы *дата · время · запрос · recap*: что и зачем менялось. Источник для рубрики build in public и постов-ретроспектив.

Это не «сгенерированный сайт»: тексты и решения — авторские, ИИ помогает со структурой, рефакторингом и рутиной.
